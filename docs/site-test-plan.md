# Site Test Plan

هذا الدليل مخصص لاختبار البوت على المواقع الخمسة التالية:

1. `alazab.com`
2. `brand-identity.alazab.com`
3. `laban-alasfour.alazab.com`
4. `luxury-finishing.alazab.com`
5. `uberfix.alazab.com`

## Before You Start

1. تأكد أن `docker compose ps` يظهر كل الخدمات `healthy`.
2. تأكد أن `http://127.0.0.1:8000/health` يرجع `status: ok`.
3. افتح الموقع المستهدف من المتصفح عبر `HTTPS`.
4. اسمح للمتصفح بالوصول إلى الميكروفون عند اختبار الصوت.

## Global Checks

نفذ هذه المراجعة على كل موقع:

1. افتح الواجهة وتأكد أن الهيدر يظهر `عزبوت (AzaBot)`.
2. تأكد أن تبويبي `محادثة نصية` و`محادثة صوتية` ظاهرين.
3. تأكد أن زر الإرفاق ظاهر في أسفل الواجهة.
4. تأكد أن زر تحميل المحادثة يعمل وينزل ملف `txt`.
5. تأكد أن فتح وإغلاق القائمة الجانبية يعمل.
6. تأكد أن المحادثة السابقة تبقى محفوظة بعد تحديث الصفحة.

## Site-Specific Expectations

### `alazab.com`

1. افتح البوت.
2. تأكد أن الرسائل السريعة عامة للشركة.
3. أرسل: `ما هي خدمات الشركة؟`
4. المتوقع: رد عام عن خدمات مجموعة العزب أو fallback واضح.
5. ارفع ملفًا باسم عرض-سعر أو ملف نصي بسيط.
6. المتوقع: تأكيد استلام الملف.
7. سجّل رسالة صوتية قصيرة.
8. المتوقع: تفريغ النص أو تأكيد استلام الصوت.

### `brand-identity.alazab.com`

1. افتح البوت.
2. تأكد أن الرسائل السريعة مرتبطة بالهوية البصرية.
3. أرسل: `أريد هوية بصرية لمعرض جديد`
4. المتوقع: توجيه مناسب لخدمات الهوية أو fallback واضح.
5. ارفع ملف شعار أو ملف نصي.
6. المتوقع: تأكيد استلام الملف.
7. سجّل رسالة صوتية عن تجهيز الفرع.
8. المتوقع: تفريغ النص أو تأكيد الاستلام.

### `laban-alasfour.alazab.com`

3. أرسل: `أحتاج كميات جملة من الخامات`
4. المتوقع: رد مناسب للتوريدات أو fallback واضح.
5. ارفع ملف طلب توريد.
6. المتوقع: تأكيد استلام الملف.
7. سجّل رسالة صوتية عن طلب توريد.
8. المتوقع: تفريغ النص أو تأكيد الاستلام.

### `luxury-finishing.alazab.com`

1. افتح البوت.
2. تأكد أن الرسائل السريعة مرتبطة بالتشطيبات.
3. أرسل: `أريد عرض سعر تشطيب شقة`
4. المتوقع: رد مناسب للتشطيبات أو fallback واضح.
5. ارفع ملف مواصفات أو صورة تشطيب.
6. المتوقع: تأكيد استلام الملف.
7. سجّل رسالة صوتية عن المساحة ونوع التشطيب.

### `uberfix.alazab.com`

1. افتح البوت.

3. أرسل: `أريد طلب صيانة تكييف`
4. المتوقع: رد مناسب للصيانة أو fallback واضح.
5. ارفع صورة عطل أو ملف نصي.
6. المتوقع: تأكيد استلام الملف.
7. سجّل رسالة صوتية تشرح العطل.
8. المتوقع: تفريغ النص أو تأكيد الاستلام.

## Pass Criteria

1. الواجهة تفتح وتغلق بدون أخطاء.
2. كل موقع يرسل `site_host` الصحيح ويختار السياق المناسب.
3. النص يعمل على `/chat`.
4. الملفات تعمل على `/chat/upload`.
5. الصوت يعمل على `/chat/audio`.
6. تحميل نسخة المحادثة يعمل.
7. لا يظهر للمستخدم صمت أو فشل غير معالج.

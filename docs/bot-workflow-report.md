# تقرير آلية عمل البوت

## الهدف

البوت هو قناة تشغيل ومحادثة موحدة لمواقع مجموعة العزب، وليس نظام طلبات منفصل. دوره استقبال رسائل العملاء من الواجهة أو القنوات، تمريرها إلى Rasa، ثم تنفيذ الإجراءات المطلوبة عبر طبقة backend آمنة.

## مسار المحادثة

1. المستخدم يفتح واجهة `azabot-prod` من الموقع العام.
2. الواجهة ترسل الرسالة إلى FastAPI على endpoint `/chat`.
3. FastAPI يحدد البراند حسب المسار أو الدومين، مثل `/uberfix`.
4. FastAPI يرسل الرسالة إلى Rasa REST webhook.
5. Rasa يحدد النية أو التدفق المناسب.
6. عند احتياج تنفيذ فعلي، Rasa يستدعي Actions server.
7. Actions server يستدعي backend المحلي حسب البراند.
8. الرد يرجع من Rasa إلى FastAPI ثم إلى الواجهة.

## مسار UberFix

عند طلب صيانة أو متابعة طلب:

1. المستخدم يكتب طلب الصيانة أو رقم التتبع.
2. Rasa يستدعي أكشن UberFix.
3. الأكشن يستدعي `/uberfix/bot-gateway` داخل FastAPI.
4. Bot Gateway يتحقق من مفتاح API.
5. Bot Gateway ينفذ العملية على PostgreSQL.
6. يتم تسجيل العملية في `api_gateway_logs`.
7. يتم تسجيل الأثر التشغيلي في `audit_logs`.
8. يرجع رقم التتبع أو حالة الطلب للمستخدم.

## الطلبات والتتبع

- المفتاح الحقيقي لكل طلب هو `UUID`.
- رقم مثل `MR-26-01045` مخصص للعرض والتتبع فقط.
- النظام لا يعتمد على رقم تسلسلي كهوية تشغيلية.
- هذا التصميم مناسب لتوسع فروع أو محلات كثيرة لأن كل سجل مستقل بمفتاح UUID.

## العمليات المدعومة في Bot Gateway

- `create_request`: إنشاء طلب صيانة.
- `check_status`: متابعة حالة الطلب.
- `get_request_details`: جلب تفاصيل طلب.
- `update_request`: تعديل بيانات مسموحة للطلب.
- `cancel_request`: إلغاء طلب قبل مراحل التنفيذ النهائية.
- `add_note`: إضافة ملاحظة عميل.
- `list_services`: عرض خدمات الصيانة.
- `list_categories`: عرض الفئات من قاعدة البيانات.
- `get_branches`: عرض الفروع.
- `find_nearest_branch`: أقرب فرع حسب الموقع.
- `collect_customer_info`: حفظ سياق بيانات العميل.
- `list_technicians`: عرض الفنيين.
- `assign_technician`: تعيين فني يدوي أو تلقائي بسيط.
- `get_quote`: استقبال طلب عرض سعر للمراجعة.

## التشغيل الصوتي

- رفع صوت المستخدم يتم عبر `/chat/audio`.
- الخادم يحفظ الملف مؤقتاً داخل uploads.
- يتم تحويل الصوت إلى نص عبر OpenAI transcription عند وجود `OPENAI_API_KEY`.
- النص الناتج يمر إلى Rasa مثل أي رسالة عادية.
- تحويل رد البوت إلى صوت يتم عبر `/chat/tts`.

## القنوات

- واجهة الموقع: `/chat`.
- رفع ملفات: `/chat/upload`.
- صوت: `/chat/audio` و `/chat/tts`.
- WhatsApp/Messenger: `/webhook/meta`.
- Telegram: `/webhook/telegram`.
- UberFix Gateway: `/uberfix/bot-gateway`.

## نقطة مهمة

البوت لا يكتب مباشرة في قاعدة البيانات من الواجهة. الكتابة تتم عبر FastAPI Bot Gateway حتى تكون المصادقة، التسجيل، والتحكم في الصلاحيات في طبقة واحدة.
